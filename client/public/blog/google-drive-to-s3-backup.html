How to Back Up Google Drive to Amazon S3

Last updated: July 2026 · 5 min read

Google Drive is convenient for collaboration and daily work, but it's not a backup solution. If files get accidentally deleted or your account gets compromised, Drive doesn't protect you from data loss. Keeping a copy in Amazon S3 gives you a durable, independent backup that you fully control.

This guide covers how to back up Google Drive to S3 — including how to automate it so it runs without any manual work.

What you need

  • A Google Drive account with the files you want to back up
  • An AWS account with an S3 bucket already created
  • An AWS IAM user with access keys (Access Key ID and Secret Access Key) that has read/write access to the bucket
  • A TERA account (free)

Setting up the backup

  1. Create a free account at mytera.app
  2. Go to Integrations and connect your Google Drive account
  3. In the same section, connect Amazon S3 — you'll need your Access Key ID, Secret Access Key and region
  4. Go to Tasks and create a new task
  5. Set the source to your Google Drive folder and the destination to your S3 bucket (you can specify a prefix/folder path within the bucket)
  6. Choose a schedule — daily is recommended for a backup use case
  7. Save the task. TERA will run the first transfer immediately and then on your schedule
Tip: Create a dedicated S3 bucket for your Drive backups with a name like my-drive-backup. This keeps the backup separate from other S3 data and makes it easy to set specific retention policies.

What gets transferred

On each run, TERA checks for files in your Google Drive folder that don't yet exist in S3 and transfers them. Files already in S3 are skipped. Google Workspace files (Docs, Sheets, Slides) are exported to standard formats (docx, xlsx, pptx) before being stored in S3.

S3 storage costs

S3 Standard storage costs approximately $0.023 per GB per month. For a 10GB Drive backup that's about $0.23/month — very cheap for a reliable off-site backup. You can also use S3 Glacier for even lower costs if you rarely need to access the backup.

One-time transfer vs. ongoing backup

If you just want to export your Drive to S3 once, you can do a manual transfer instead of a scheduled task — open the Cloud Explorer, set Drive on one side and S3 on the other, and transfer the folder. For ongoing backups, the scheduled task is the right approach.
